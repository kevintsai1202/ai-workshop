## 單元 1｜AI 基礎概念與多工具定位（10:00 ~ 12:00）

### 1-1 學習目標

- 用一句話說明「生成式 AI」是什麼。
- 分得清「**模型**（Gemini / GPT）」與「**工具**（Gemini App / ChatGPT / NotebookLM）」的差別。
- 知道 Gemini 與 ChatGPT 兩大模型的差異與適用場景。
- 知道 Gemini / NotebookLM / Antigravity / Codex 各自的定位差異。
- 完成 4 個工具的帳號註冊與登入。

### 1-2 核心觀念講授

#### A. 生成式 AI 的「本質」：機率預測機

別把 AI 當神，它是「**接龍機**」：你給它一段話，它預測下一個字最可能是什麼，然後一直接下去。

- **它擅長**：歸納、改寫、模仿風格、找模式
- **它不擅長**：精準計算、即時資訊、判斷對錯

> **講師舉例**：
>
> - 你問它「2+2=?」它答對不是因為它會算，是因為網路上有一兆個「2+2=4」。
> - 你問它「我們公司上週的營業額？」它就會掰，因為它沒有資料。
> - **結論**：AI 是助手，不是答案機。永遠人在迴圈裡（Human in the Loop）。

#### B. 模型 vs 工具：別把兩個層級搞混（重要！）

學員最常見的誤解：「我用 ChatGPT」「我用 Gemini」其實同時在指**兩件事**——

| 層級                         | 你看到的                                         | 實際是什麼                               | 比喻 |
| ---------------------------- | ------------------------------------------------ | ---------------------------------------- | ---- |
| **模型**（Model）      | GPT-5、Gemini 2.5 Pro、Claude 4                  | AI 的「大腦」：負責思考與生成            | 引擎 |
| **工具**（Tool / App） | ChatGPT App、Gemini App、NotebookLM、Antigravity | 包裝模型的「介面」：負責互動、上傳、整合 | 車身 |

> **講師舉例**：
>
> - **同一台引擎**（GPT 模型）可以裝在不同車身：ChatGPT 網頁、ChatGPT App、Microsoft Copilot、Notion AI…
> - **同一台車身**（NotebookLM）裡裝的是 Gemini 模型；**Antigravity** 則可以選擇底層用 Gemini 或其他模型。
> - 所以你聽到「我用 ChatGPT」**不等於**「我用 GPT-5」——可能是 GPT-4o、可能是 GPT-5、甚至是免費版的 GPT-4o-mini。

#### C. 兩大主流模型對比：Gemini vs ChatGPT（GPT 系列）

> ⚠️ 本表為授課當下版本對比，模型版本更新快，每月確認一次最新狀況。

| 比較維度                           | Gemini（Google）                                       | ChatGPT / GPT（OpenAI）                            |
| ---------------------------------- | ------------------------------------------------------ | -------------------------------------------------- |
| **背後公司**                 | Google                                                 | OpenAI（微軟為主要投資者）                         |
| **最新模型**                 | Gemini 3.1 Pro / Flash                                | GPT-5.5                                            |
| **多模態（圖/影/音）**       | ⭐**原生多模態**，圖片影片直接懂, 能直接升成影片 | 讀取部分略遜Gemini, 不過最新的生圖模型已追過Gemini |
| **長文處理**                 | ⭐ Context window 高達 1M~2M tokens                    | 通常 128K~400K，相對短                             |
| **推理能力**                 | 強（但傳統印象稍弱於 OpenAI）                          | ⭐ o1/o3/GPT-5 系列推理表現頂尖                    |
| **與 Google Workspace 整合** | ⭐ Gmail、Docs、Drive、Calendar 直接呼叫               | 需透過第三方插件                                   |
| **免費版額度**               | ⭐ 大方（Gemini App 幾乎免費吃到飽）                   | 較緊（免費版額度低、會降模型）                     |
| **企業/商用條款**            | 商用方案下不用對話訓練模型                             | 商用方案下不用對話訓練模型                         |
| **中文（繁中）表現**         | ⭐ 近年明顯進步，台灣語境自然                          | 仍不錯，但偶有「中國用語」傾向                     |
| **生態系/插件**              | 較少（但 Google 自家服務深度整合）                     | ⭐ GPTs / 插件 / 開發者生態最豐富                  |
| **本課程使用**               | ⭐**主力工具**                                   | 補充工具（部分學員可能已有訂閱）                   |

> **講師重點口訣**：
>
> - **Gemini**：「**長文 + 影音 + 與 Google 整合**」 → 行政人員的甜蜜點
> - **ChatGPT**：「**深度推理 + 生態系**」 → 程式 / 數據分析 / 已有 ChatGPT Plus 訂閱者繼續用

#### D. 為什麼本課程主推 Gemini？（學員必問）

1. **免費版夠用**：學員不需付費就能完整跑完 4 天課程。
2. **NotebookLM 是 Google 自家**：與 Gemini 模型無縫銜接，行政場景下無可替代。
3. **與 Google Workspace 深度整合**：90% 行政人員天天用 Gmail / Docs / Drive。
4. **中文長文處理能力強**：適合會議紀錄、長公告等台灣行政常見場景。

> **不代表 ChatGPT 不好**——若你公司已有 ChatGPT Plus，**95% 的提示詞技巧可直接通用**，只是部分工具（NotebookLM、Antigravity）會回到 Google 生態。

#### E. 模型版本更新節奏：給學員的長期心態

- **每 3-6 個月會有大版本**（例：Gemini 3.0 → 3.1；GPT-5.4 → GPT-5.5）。
- **不要追版本**，追**能力**：你需要的是「能讀長文、能看圖、能生圖片影片」這些能力，而非特定版本號。
- **本課程教的是『工作流』**——**模型升級也通用**，是真正的長期投資。

#### F. 為什麼要學「多工具」？因為一招打天下會出事

| 工具                  | 一句話定位                                     | 行政場景對應                               |
| --------------------- | ---------------------------------------------- | ------------------------------------------ |
| **Gemini**      | 通用對話與內容生成（Google 的 ChatGPT）        | 寫公告、改文案、長文摘要                   |
| **NotebookLM**  | **以你提供的資料為唯一來源**的 AI 筆記本 | 會議錄音轉摘要、發票辨識、企業內部資料整理 |
| **Antigravity** | 桌面型 AI 代理（IDE-like），可幫你自動跑流程   | 把多步驟工作打包成 Skill 自動執行          |
| **Codex**       | 偏程式碼與腳本生成的 AI 工具                   | 能在電腦處理的工作都能交給它               |

> **講師舉例（暖光咖啡）**：
>
> - 阿凱店長想寫一篇「夏季限定特調」公告 → 用 **Gemini**
> - 阿凱要把 30 分鐘員工會議錄音整理成會議紀錄 → 用 **NotebookLM**（因為它只看你的錄音、不會亂掰）
> - 阿凱想做一個「拍張發票照片就自動轉 Excel」的工具 → 用 **Antigravity + Skill**
> - 阿凱要寫一個線上預約系統 → 用 **Codex**

#### G. 為什麼 NotebookLM 是行政人員的神器（重點停留）

NotebookLM 與 Gemini 的關鍵差異：**它只會根據你給它的資料回答**。

- 你給它 5 份文件，它就只看這 5 份。
- 它不會幻想、不會引用網路上的資訊。
- 對行政與財務工作這是**無可取代的優點**：因為你不能讓 AI 幻想出公司沒有的數據。
- **也是優秀的個人學習工具**：把講義／教科書／論文丟進去，就能對話問答、產生摘要與 podcast — 本課僅點到為止，有興趣的學員可課後自行探索。

## 單元 2 上半｜提示詞入門（13:00 ~ 14:00）

### 2-1 學習目標

- 能說出提示詞的「四要素」：角色、任務、格式、限制。
- 看到一段爛提示詞能立刻指出「缺哪一塊」。
- 能依公告 / 通知 / 會議紀錄三種文件類型寫出對應提示詞。

### 2-2 核心觀念講授：提示詞四要素（RTFC）

> **R**ole（角色）+ **T**ask（任務）+ **F**ormat（格式）+ **C**onstraint（限制）

#### 對比示範（一定要做）

**❌ 爛提示詞**：

> 「幫我寫個咖啡廳的公告」

**結果**：AI 給你一個八股、不知道用在哪、長度不對、語氣也不對的東西。

**✅ 好提示詞（RTFC 完整版）**：

> 你是一家信義區單店咖啡廳「暖光咖啡」的店長【R】。
> 請幫我寫一篇「7 月起夏季限定手沖咖啡上市」的公告【T】。
> 格式：第一段感性開場 30 字，第二段三點式條列產品特色，第三段 CTA 行動呼籲【F】。
> 限制：總字數 200 字內，語氣溫暖親切，避免使用「敬請期待」「即將推出」等套話【C】。

> **講師備忘**：把兩個結果**並排投影**讓學員親眼看到差異，比講十遍道理有效。

#### 進階觀念：給 AI「範例」效果遠大於給「規則」

- 想要 AI 寫出某種風格 → **直接貼一篇你過去寫過、覺得不錯的範例**給它，比解釋風格更快。
- 這叫做 **Few-shot Prompting**（少樣本提示）。

### 2-3 講師示範腳本：3 種行政文件提示詞（45 分鐘）

#### 示範 A｜公告類（暖光咖啡）

```
角色：你是「暖光咖啡」店長，需要對顧客發布訊息。
任務：撰寫一篇「7/1 起店休調整為週二公休」的公告。
格式：FB 貼文格式，含 1 個 emoji 開頭、3 段內容、結尾感謝詞。
限制：150 字內，語氣親切但專業，不要使用驚嘆號超過 2 個。
```

#### 示範 B｜通知類（綠野選物）

```
角色：你是「綠野選物」總部行政主管，要對 3 家門市同仁發內部 Email 通知。
任務：通知 8/15 全員教育訓練，地點台北總部，時間 14:00-17:00。
格式：Email 標準格式（主旨 + 收件對象 + 內文 + 行前須知 4 點 + 簽名）。
限制：正式語氣、條列清楚，內文 250 字內。
```

#### 示範 C｜會議紀錄類（暖光咖啡）

```
角色：你是會議紀錄專員。
任務：根據以下會議口述內容，整理成正式會議紀錄。
[此處貼上錄音逐字稿或口述稿]
格式：會議紀錄五段式（時間/出席/議題/決議/待辦）。
限制：決議與待辦務必標明負責人與期限；原文沒講的不要自己補。
```

> **講師備忘**：示範 C 最容易展現 AI 的「邊界」——強調「原文沒講的不要自己補」這條限制，並讓 AI 跑兩次（有 / 沒有這條限制），看差異。

### 2-4 學員小練習：救救爛提示詞

- 給 3 段「爛提示詞」，請學員 5 分鐘內補成 RTFC 完整版。
- 抽 2 位上台分享改寫結果。

## 單元 2 下半｜行政應用實作（14:00 ~ 15:30）

### 實作 1｜口語改寫成正式公告（45 分鐘）

#### 任務說明

- 學員拿到一段「老闆口氣的口語錄音稿」（講師提供）。
- 用 Gemini，套用上午學的 RTFC，改寫成一篇可直接貼出去的公告。

#### 範例素材（口語原稿）

> 「欸大家，那個…我們從下個月開始啊，禮拜二要公休啦，因為人手不夠，然後…那個冷氣也要清，所以就那天嘛，請大家跟客人說一下，FB 也 PO 一下，跟熟客講清楚，不要被罵。」

#### 預期產出

- 一份 200 字內、可直接貼到 FB 或 Line 群組的公告文。

#### 評核點

| 評核項目               | 標準                         |
| ---------------------- | ---------------------------- |
| 是否完整含 RTFC 四要素 | 缺一項扣分                   |
| 是否消除口語廢話       | 「那個」「然後」全部移除     |

| 是否符合語氣要求       | 親切但不失專業               |

## 單元 3｜Gem 介紹與實作（15:30 ~ 16:30，60 分鐘）

### 3-2 Gem 是什麼？（3 分鐘）

> **一句話定義**：**Gem = Gemini App 內的「客製化小助手」**，把你常用的提示詞、角色、知識來源，封裝成一個可一鍵呼叫的工具。

### 3-3 Gem 的三大來源（5 分鐘）

> Gem 不是只有「自己做」這條路。實際上有三種來源，**對應三種使用者成熟度**。

| --------------------------------- | --------------------------------------------------------------- | ---------------------- | ------------------------------------------------------ |

| 2️⃣**自訂 Gem**           | Gem 管理工具 → 新增 Gem                                        | **行政應用主力** | **下面 3-4 段親手做**                            |
| 3️⃣**Google Labs 的 Gem** | [https://labs.google](https://labs.google)                         | 進階嘗鮮者             | **Day 4 開場深入**（Antigravity 也是 Labs 出身） |
