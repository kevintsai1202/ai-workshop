## 使用方式

### A. 講師端

**已預先建置的 PDF**：直接使用 [pdf/](pdf/) 目錄下四份檔案，上傳到 NotebookLM 作為示範來源。

**重新建置 PDF**（修改 `.md` 後）：

```powershell
pwsh -File _build/build-pdf.ps1
```

* 工具鏈：`pandoc` (md→html) + `Microsoft Edge --headless --print-to-pdf` (html→pdf)
* 樣式檔：[_build/style.css](_build/style.css)（A4 版面、深藍企業色、宋體中文優化）
* 加 `-KeepHtml` 參數可保留中介 HTML 供除錯

**課堂運用**：

1. 上傳 `pdf/*.pdf` 到 NotebookLM。
2. 引導學員觀察「AI 引用條文」的精準度。

### B. 學員端

1. 學員可下載 `.md` 直接拖入 NotebookLM。
2. 用「課程內容」中提供的提示詞範本，實際生成公告／客訴回覆／合規檢查表。
3. 將產出貼回自己的提示詞庫，作為 Day 4 個人 Skill 的素材。

## 文件設計原則（給課程開發團隊）

這些文件刻意內含以下「教學陷阱」，方便講師展示 AI 的精準度：

| 文件 | 教學陷阱 | 對應提示詞會問什麼 |
| --- | --- | --- |
| 員工差勤辦法 | 第十一條「國定假日加班費**加倍發給**」 | 「端午節加班費怎麼算？」 |
| 退換貨政策 | 第六條「拆封濾掛**一律不退**」+ 第十條折衷補償 | 「客訴：濾掛喝了一包說太酸要全退」 |
| 費用報支辦法 | 第四條「誤餐費**上限 150 元**」 | 「便當 360 元能不能報？」 |
| 客訴處理 SOP | LAST 四步驟、L3 分級 | 「異物事件怎麼回應？」 |
| 新品上架辦法 | 第八條「IG 限 150 字 + 5 個 hashtag」 | 「請改寫成 IG 貼文」（檢驗 AI 是否守字數） |
| 食材庫存辦法 | 第十一條「咖啡豆烘焙後 14 日為最佳賞味期」 | 「6/1 烘焙的豆子能不能 6/20 賣？」 |
| 員工訓練手冊 | 跨章節：第五章薪資 + 第十章績效（D 等第連動） | 「連 2 季 D 會怎樣？」（測 NotebookLM 跨章節跳查） |
| FAQ 官方版 | 對照課程輔助文件 10 條原話 | 學員產出與此版差異超過 30% 須重做 |

## 內容免責聲明

* 所有公司名稱、人物、聯絡方式、統編、條款內容均為**虛構**，僅供教學示範。
* 文件中之法規條文（勞基法、消保法）已盡量符合台灣現行法令，但**不得作為任何法律或人事決策依據**。
* 真實公司導入時，請務必由人資、法務、財會專業人員另行審閱。
